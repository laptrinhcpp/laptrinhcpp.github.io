2.1 Giới thiệu về hàm

Ở chương trước, ta chỉ biết qua hàm (function) là một tập hợp các câu lệnh. Ở chương này, ta sẽ “nâng cấp” định nghĩa đó lên một chút. Hàm là tập hợp các câu lệnh, để làm một công việc cụ thể nào đó và nó có thể tái sử dụng nhiều lần.

Vì sao ta cần đến hàm?

Như bạn đã biết, tất cả chương trình C++ đều cần một hàm đặc biệt là hàm main. Trình biên dịch sẽ biên dịch dòng đầu tiên của hàm main trước và đến dòng cuối thì chương trình dừng lại. Do đó, bạn hoàn toàn có thể đặt tất cả code bên trong hàm main và chương trình vẫn chạy tốt. Tuy nhiên, khi chương trình của bạn bắt đầu dài ra thì việc làm trên sẽ làm cho việc quản lý code trở nên khó khăn, mọi thứ trở nên rối rắm. Lúc này, chúng ta cần đến hàm. Hàm cho chúng ta khả năng tách code thành những phần nhỏ, khiến code dễ đọc hơn, bạn có thể dùng nó lại nhiều lần, bạn cũng có thể tách nó riêng ra để test rồi mới đưa vào chương trình.

Cách thức hoạt động của hàm

Hàm được tạo ở ngoài hàm main, nhưng ở trên mình đã nói là trình biên dịch biên dịch từ dòng đầu tiên của hàm main đến dòng cuối cùng hàm main. Vậy làm sao trình biên dịch có thể biên dịch các hàm ngoài hàm main? Giả sử bạn đang đọc một cuốn sách, bạn đọc đến trang nào đó, ví dụ trang 27. Đột nhiên bạn nhớ mình phải đi phơi đồ, trước khi đi phơi đồ bạn sẽ đánh dấu trang đang đọc. Khi phơi đồ xong, bạn lật lại trang đã đánh dấu và tiếp tục đọc tiếp. Trình biên dịch cũng hoạt động như vậy; trong hàm main, nó sẽ “đánh dấu” vị trí mà hàm được gọi tại đó. Khi trình biên dịch đến vị trí gọi hàm, nó sẽ “đánh dấu” vị trí đó lại rồi tìm đến nơi hàm đó được định nghĩa để biên dịch. Sau khi biên dịch hàm đó xong nó sẽ quay lại vị trí đã “đánh dấu” trong hàm main và tiếp tục biên dịch tiếp.

#include <iostream>

void print()
{
    std::cout << "This is function.\n";
}

int main()
{
    std::cout << "This is the first statement of main function.\n";
    
    print();     // Gọi hàm. Trình biên dịch tìm đến hàm print() và biên dịch code trong đó
    // Trình biên dịch quay trở lại hàm main
    std::cout << "This is the last statement of main function.\n";
}

Chương trình trên cho ra output:

This is the first statement of main function.
This is function.
This is the last statement of main function.

Lưu ý: Sau tên hàm luôn phải có ().

Khi biên dịch chương trình trên, trình biên dịch bắt đầu từ dòng 10 nên in “This is the first statement of main function.”. Sau đó, nó gặp câu lệnh gọi hàm print() nên nó tìm đến nơi hàm print() được tạo ra, tức dòng 3 và chạy tất cả những gì được viết trong hàm. Do đó, nó in ra “This is function.”. Sau khi chạy xong code trong hàm print(), nó quay lại hàm main ngay sau vị trí gọi hàm print() và tiếp tục đến hết hàm main, in ra “This is the last statement of main function.”. Chương trình kết thúc.

#include <iostream> 

void print() 
{ 
    std::cout << "This is function.\n"; 
} 

int main() 
{ 
    std::cout << "This is the first statement of main function.\n"; 
    
    print();    // gọi lần 1
    print();    // gọi lần 2

    std::cout << "This is the last statement of main function.\n"; 
}

Chương trình trên cho ra output như sau:

This is the first statement of main function. 
This is function. 
This is function. 
This is the last statement of main function.

Ở chương trình trên thì ta ít thấy lợi ích này. Tuy nhiên, nếu hàm đó gồm vài chục câu lệnh thì bạn đã tiết kiệm rất nhiều thời gian và dòng code rồi đấy.

#include <iostream>
 
void doB()
{
    std::cout << "In doB()\n";
}
 
 
void doA()
{
    std::cout << "Starting doA()\n";
 
    doB();
 
    std::cout << "Ending doA()\n";
}
 
int main()
{
    std::cout << "Starting main()\n";
 
    doA();
 
    std::cout << "Ending main()\n";
 
    return 0;
}

Chương trình trên cho output như sau:

Starting main()
Starting doA()
In doB()
Ending doA()
Ending main()

Lưu ý: Bạn không được tạo hàm trong hàm main hay bất kỳ hàm nào khác. Tất cả đều phải tạo riêng.
