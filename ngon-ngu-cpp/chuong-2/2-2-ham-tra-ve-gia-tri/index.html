Chạy chương trình trên thì máy báo lỗi, không chạy được. Vì sao nhỉ? Nhìn lại chương trình, ta thấy biến num chưa được gán giá trị input, nó mang giá trị 0. Biến mà được gán giá trị người dùng nhập vào là biến input trong hàm getValueFromUser(). Vậy muốn sử dụng được giá trị input, ta phải trả nó về hàm main, để rồi gán nó cho biến trong hàm main và có thể sử dụng được.

Trả về giá trị

Khi bạn tạo một hàm, điều đầu tiên cần làm là xác định xem hàm đó có cần trả về một giá trị nào đó hay không. Để tạo hàm có trả về giá trị, ta cần làm hai việc.

Đầu tiên, xác định giá trị trả về đó có kiểu dữ liệu gì. Điều này sẽ quyết định phần trước tên hàm là gì. Lấy ví dụ là hàm main. Có thể bạn thắc mắc tại sao có chữ int trước tên hàm main. Đó chính là kiểu dữ liệu của giá trị trả về mà cụ thể chính là 0.

Thứ hai, trong hàm phải có câu lệnh trả về. Trong hàm main chính là câu lệnh “return 0;”. Sau khi trình biên dịch gặp câu lệnh này. Nó sẽ trả giá trị ra ngoài hàm và kết thúc hàm, cho dù sau return có bao nhiêu câu lệnh đi nữa cũng sẽ bị bỏ qua.

#include <iostream>
 
int returnFive()   // hàm trả về giá trị kiểu int nên có int trước trên hàm
{
    std::cout << "Before return statement.\n";
    return 5;        // câu lệnh trả về giá trị 5
    std::cout << "After return statement.\n";
}
 
int main()
{
    std::cout << returnFive() << '\n';       // int 5 ra màn hình
    std::cout << returnFive() + 2 << '\n';   // in 7 ra màn hình
  
    return 0;
}

Chương trình trên cho ra output:

Before return statement.
5
Before return statement.
7

Các bạn có thể thấy nó chỉ in dòng “Before return statement.” chứ không in “After return statement.” vì khi gặp câu lệnh return thì trình biên dịch thoát khỏi hàm, trả giá trị về hàm main.

Khi chương trình chạy, câu lệnh dòng 12 sẽ thực thi đầu tiên. Gặp câu lệnh gọi hàm returnFive(), trình biên dịch tìm đến hàm returnFive(). Chương trình in dòng “Before return statement.”, sau đó trả giá trị 5 về hàm main (bỏ qua câu lệnh còn lại) và in ra màn hình nhờ std::cout. Trình biên dịch tiếp tục gặp câu lệnh gọi hàm returnFive(). Lúc này “returnFive() + 2” chính là một biểu thức, và giá trị đầu ra của biểu thức này là 7 (5 + 2), do đó 7 được in ra màn hình.

Lưu ý: Khi bạn đã dùng hàm trả về thì bạn phải làm gì đó với giá trị trả về, nếu không thì giá trị đó coi như vô nghĩa. Và lúc đó bạn nên dùng hàm không trả về giá trị (void) mà mình sẽ nói ở dưới.

Hàm void

Ở trên mình đã nói về hàm có trả về giá trị. Còn nếu bạn không muốn trả về giá trị, hãy dùng hàm void. Thay vì để kiểu dữ liệu trước tên hàm như trên, ta thay nó bằng void. Bạn đã gặp loại hàm này ở bài trước. Bởi vì nó không trả về giá trị nên không cần câu lệnh return. Nếu bạn thêm cho nó câu lệnh return về một giá tri, trình biên dịch sẽ báo lỗi.

#include <iostream>

void print()
{
    std::cout << "Hello ";

int main()
{
    print();

    std::cout << "world.";

    return 0;
}

Chương trình trên cho output:

Hello world.

Khác với hàm có trả về, hàm void không được gán cho biến hoặc đưa nó vào std::cout, máy sẽ báo lỗi vì hàm void không trả về giá trị. Phần lớn, bạn sẽ dùng hàm void để in cái gì đó ra màn hình.

Giả sử, vì một lý do nào đó, bạn chỉ muốn trình biên dịch biên dịch một phần của hàm void và bỏ qua phần còn lại. Lúc này, bạn có thể dùng câu lệnh return. Ủa, khoan. Chẳng phải mình đã nói không dùng câu lệnh return trong hàm void sao? Đúng vậy, bạn không thể return một giá trị nhưng bạn có thể return một thứ không giá trị. Nghe có vẻ vô lý đúng không. #include <iostream>

void print()
{
    std::cout << "Line 1.\n";
    
    return;   // câu lệnh return nhưng không có giá trị trả về
    
    std::cout << "Line 2.\n";
}

int main() 
{
    print();   // gọi hàm print()
    
    return 0;
}

Chương trình trên cho output:

Line 1.

Câu lệnh return ở ví dụ trên không trả về giá trị nào cả, và nó có chức năng kết thúc hàm. Do đó, chương trình chỉ in dòng “Line 1.” và bỏ qua dòng “Line 2.”

Bây giờ chúng ta nói về hàm main nhé. Như bạn có thể thấy, hàm main bản chất là một hàm có trả về. Nhưng giá trị trả về đó – được gọi là status code (thường là 0) được gửi đi đâu, được xử dụng vào mục đích gì? Câu trả lời là giá trị đó được gửi về hệ điều hành (operating system). Mục đích để thông báo cho hệ điều hành biết chương trình có chạy thành công hay không. Việc chương trình trả về giá trị 0 có nghĩa là tất cả code đều chạy thành công đến dòng cuối cùng của hàm main. Còn nếu chương trình bị crash giữa chừng thì nó sẽ trả về giá trị khác 0.

Ở phần lớn các IDE, nó sẽ báo cho bạn biết status code sau khi kết thúc chương trình là bao nhiêu. Ví dụ dưới đây là thông báo chương trình chạy thành công.

Program finished with exit code 0

Còn đây là thông báo chương trình bị crash.

Program finished with exit code 1

Exit code kia có thể là bất cứ số nào khác 0.

Lưu ý về hàm trả về

Đối với một hàm có trả về bình thường, bắt buộc phải có giá trị trả về, tức câu lệnh return. Nếu không sẽ gây ra undefined behavior. Chỉ riêng hàm main, mặc dù là hàm có trả về nhưng bạn có thể bỏ qua câu lệnh return mà chương trình vẫn chạy bình thường. Lúc đó, chương trình sẽ mặc định hàm main có giá trị trả về là 0. Tuy nhiên, bạn không nên bỏ câu lệnh return trong hàm main, đó là thói quen không tốt.

Một hàm trả về chỉ có thể trả về một giá trị. Giá trị đó có thể là literal, giá trị của biến, giá trị của một biểu thức hoặc giá trị trả về của một hàm khác. Ví dụ:

return x + 2;
return x + y;

Ở chương trình trên, mọi thứ đều ổn. Chỉ duy nhất một khuyết điểm là lặp lại code (một trong những điều nên tránh trong lập trình). Bạn hãy nhớ phương châm tiêu biểu của lập trình đó là Don’t Repeat Yourself (DRY). Vì sao vậy? Đó là vì giả sử bạn cần sửa “Enter an integer: ” thành “Enter a number: “, bạn phải sửa hai chỗ. Đây mới là chương trình cơ bản. Bạn thử tưởng tượng chương trình có trên 1000 dòng code với 200 dòng code cần sửa được đặt ngẫu nhiên trong 1000 dòng đó. Và bạn phải tìm trong đống đó những dòng code cần sửa xem. Còn một lý do nữa là lặp lại code sẽ làm code dài ra.
