1.4 Gán biến và khởi tạo biến

Ở bài 1.3 Biến, chúng ta đã tìm hiểu qua cách định nghĩa một biến để lưu trữ giá trị. Trong bài này, ta sẽ tìm hiểu cách gán giá trị cho biến và sử dụng chúng. Mình xin nhắc lại cách định nghĩa biến: 

1
2
3
// Định nghĩa một lúc nhiều biến
int a;     // định nghĩa biến x có kiểu dữ liệu int
int x, y;  // định nghĩa hai biến x, y có kiểu dữ liệu int

Gán giá trị cho biến

Sau khi định nghĩa biến, bạn có thể gán giá trị cho nó thông qua toán tử ‘=’: 

1
2
3
int score;   // định nghĩa biến score có kiểu dữ liệu int
score = 10;  // gán giá trị 10 vào biến score
// biến score bây giờ mang giá trị 10

Toán tử ‘=’ có chức năng copy giá trị ở bên vế phải và gán giá trị đó cho biến ở vế trái nên được gọi là toán tử gán (assignment operator).

Đây là cách gán giá trị cho một biến hai lần: 

 1
 2
 3

10

#include <iostream>
 
int main()
{
  int score;  // định nghĩa biến score
  score = 10; // gán giá trị 10 vào biến score
 
  // biến score hiện có giá trị 5
 
  score = 20; // gán giá trị 20 vào biến score
 
  // biến score hiện có giá trị 7
 
  return 0;
}

Khi bạn gán 20 vào biến score, giá trị 10 sẽ bị ghi đè và mất đi. Mỗi biến chỉ có thể lưu trữ một giá trị.

Lưu ý: sau này có thể bạn sẽ gặp thêm một loại toán tử là ‘==’. Đây là toán tử so sánh và nó không có chức năng gán giá trị như toán tử ‘=’, bạn nên tránh nhầm lẫn.

Khởi tạo gán và khởi tạo trực tiếp

Gán giá trị như các ví dụ trên có mặt không tốt. Đó là cần ít nhất hai câu lệnh: định nghĩa biến và gán giá trị cho biến. Rất may, C++ cho phép bạn làm hai điều đó cùng một lúc, được gọi là khởi tạo biến (initialization).

C++ cung cấp cho bạn ba loại khởi tạo. Đầu tiên là khởi tạo gán (copy initialization) bằng toán tử ‘=’: 

1
int score = 10; // khởi tạo biến score có giá trị 10

Cách hai là khởi tạo trực tiếp (direct initialization) bằng dấu ‘()’: 

1
int score(10);  // khởi tạo biến score có giá trị 10

Đối với các kiểu dữ liệu đơn giản như int, hai cách trên là như nhau. Tuy nhiên, đối với kiểu dữ liệu phức tạp như struct, class thì khởi tạo trực tiếp sẽ cho hiệu suất tốt hơn.

Khởi tạo đồng nhất

Rất tiếc, khởi tạo trực tiếp không phải lúc nào cũng có thể dùng được. Do đó, C++ (từ 11 trở lên, các phiên bản thấp hơn không hỗ trợ) cung cấp thêm một loại khởi tạo nữa gọi là khởi tạo đồng nhất (uniform initialization). Loại khởi tạo này dùng dấu ‘{}’ thay vì ‘()’ như khởi tạo trực tiếp: 

1
int score{10};  // khởi tạo biến score có giá trị 10

Khởi tạo đồng nhất mà bỏ trống ‘{}’ thì được gọi là zero initialization. Biến đó sẽ mang giá trị 0: 

1
int score{};  // khởi tạo biến score có giá trị 0

Khởi tạo đồng nhất có một ưu điểm. Đó là nó không cho phép bạn đưa giá trị có kiểu dữ liệu khác với kiểu dữ liệu của biến vào biến khi khởi tạo. Do đó, nó sẽ báo lỗi khi bạn khởi tạo biến với giá trị không hợp lệ. 

1
2
3
4
5
6
int a = 6.9;
int b(6.9);
/* 6.9 là số thập phân, khác số nguyên
 * nhưng máy sẽ không báo lỗi
 * biến a và b sẽ đều mang giá trị 6, nguyên nhân vì sao như vậy sẽ đề cập ở các bài sau */
int c{6.9};  // trường hợp này máy sẽ báo lỗi

Đó là một điều tốt vì nếu bạn không cẩn thận, bạn có thể gán nhầm giá trị, dẫn tới sai sót số liệu. Lấy ví dụ trường hợp trên 6.9 chỉ còn 6. Trình biên dịch sẽ báo lỗi và bạn sẽ biết mình đã gán giá trị không hợp lệ.

Nếu bạn dùng phiên bản C++ 11 trở lên. Bạn nên dùng khởi tạo đồng nhất để khởi tạo biến.

Khởi tạo đa biến

Ở bài trước, ta biết có thể định nghĩa nhiều biến cùng lúc với điều kiện cùng kiểu dữ liệu. Và khởi tạo biến cũng vậy, bạn có thể khởi tạo nhiều biến cùng lúc: 

1
2
3
int a = 5, b = 6;    // khởi tạo gán
int c( 7 ), d( 8 );  // khởi tạo trực tiếp
int e{ 9 }, f{ 10 }; // khởi tạo đồng nhất (nên dùng)

Lưu ý cách viết sau: 

1
2
3
int a, b = 5;      // sai (a không được khởi tạo)
 
int a = 5, b = 5;  // đúng

Ở cách viết thứ nhất trong ví dụ trên, có thể trình biên dịch sẽ báo lỗi hoặc không. Nhưng chương trình có thể bị crash bất cứ lúc nào. Mình sẽ nói về vấn đề sử dụng biến không khởi tạo như thế sẽ có ảnh hưởng như thế nào ở các bài sau.
