3.10 Giới thiệu câu lệnh if

Giả sử trong trường hợp bạn đi chợ và nghĩ rằng "nếu giá thịt lợn giảm thì mình sẽ mua". Đây là một câu điều kiện, có nghĩa là bạn sẽ làm một số hành động (mua) nếu các điều kiện cho trước đúng (giá thịt lợn giảm).

Trong lập trình cũng có các câu điều kiện như vậy. Câu điều kiện đơn giản nhất trong C++ là câu lệnh if (if statement). Câu lệnh if cho phép chúng ta thực hiện một hoặc một số dòng code nếu một số điều kiện cho trước đúng. Câu lệnh if đơn giản nhất có dạng sau:

if (dieu_kien) cau_lenh;

Để cho dễ đọc, người ta thường viết thành:

if (dieu_kien)
 cau_lenh;

dieu_kien (condition) là một biểu thức cho ra giá trị boolean.

Nếu giá trị của dieu_kien là true thì cau_lenh sẽ được thực thi. Nếu giá trị của dieu_kien là false thì cau_lenh sẽ được bỏ qua.

#include <iostream>
 
int main()
{
    std::cout << "Enter an integer: ";
    int x {};
    std::cin >> x;
 
    if (x == 0)
        std::cout << "The value is zero\n";
 
    return 0;
}

Chạy thử chương trình trên:

Enter an integer: 0
The value is zero

Đầu tiên, người dùng nhập giá trị 0 vào. Giá trị 0 được lưu vào biến x. Tiếp theo chương trình xét điều kiện x == 0. Toàn tử == trả về true nếu x bằng 0 và trả về false nếu x khác 0. Vì x = 0 nên nó trả về true, do đó câu lệnh std::cout << The value is zero\n; được thực thi và in ra The value is zero.

Còn đây là output khi ta nhập 5:

Enter an integer: 5

Ở trường hợp này, điều kiện x == 0 sai nên nó trả về giá trị false. Kết quả là câu lệnh sau if bị bỏ qua và chương trình không in gì thêm.

Lưu ý: Cách viết câu lệnh if trên chỉ áp dụng để thực thi một câu lệnh sau if. Còn cách để tạo câu lệnh if để thực thi nhiều câu lệnh thì các bạn sẽ được học ở bài sau.

Cả hai chương trình trên đều hoạt động tốt. Tuy nhiên, nó khá là phức tạp so với những gì chúng ta cần. Thay vào đó, chúng ta có thể dùng một dạng mở rộng của câu lệnh if là if-else. If-else có dạng như sau:

if (dieu_kien)
    cau_lenh_true;
else
    cau_lenh_false;

Nếu giá trị của dieu_kien là true thì cau_lenh_true sẽ được thực thi và cau_lenh_false sẽ được bỏ qua, nếu dieu_kien là false thì ngược lại.

Câu lệnh if nối tiếp nhau

Đôi khi chúng ta sẽ cần kiểm tra nhiều hơn 1 điều kiện. Ví dụ: nếu số đã cho lớn hơn 0 thì nó là số dương, nhỏ hơn 0 là số âm, còn không thì nó bằng 0. Lúc này, ta sẽ nối các câu lệnh if lại thành một chuỗi có mối liên kết với nhau.

#include <iostream>
 
int main()
{
    std::cout << "Enter an integer: ";
    int x{};
    std::cin >> x;
 
    if (x > 0)
        std::cout << "The value is positive\n";
    else if (x < 0)
        std::cout << "The value is negative\n";
    else 
        std::cout << "The value is zero\n";
 
    return 0;
}

Ở chương trình trên, bạn cần chú ý đến các điều sau. Toàn tử bé hơn (<) được dùng để kiểm tra xem giá trị ở vế trái có nhỏ hơn giá trị ở vế phải không; nếu nhỏ hơn, nó sẽ trả về giá trị true, còn không thì trả về false. Tương tự đối với toán tử lớn hơn (>) nhưng nó hoạt động ngược lại so với toán tử bé hơn. Khi muốn nối các câu lệnh if lại với nhau, câu lệnh đầu tiên ta dùng từ khoá if, các câu lệnh điều kiện tiếp theo dùng else if, bạn có thể viết bao nhiêu else if tuỳ thích. Ở điều kiện cuối cùng, nếu nó là phủ định duy nhất của tất cả điều kiện ở trên (tức là ngoài các trường hợp trên) thì ta dùng else. Chẳng hạn như ví dụ ở trên, một số thì có ba khả năng: lớn hơn 0, nhỏ hơn 0 và bằng 0. Nếu không lớn hơn 0 thì có thể nó nhỏ hơn 0, nếu nó không nhỏ hơn 0 thì chắc chắn nó bằng 0, ngoài ra không còn khả năng khác. Đó là lý do ta dùng else ở câu lệnh cuối cùng.

Dưới đây là một vài ví dụ chạy chương trình trên:

Enter an integer: 4
The value is positive
Enter an integer: -3
The value is negative
Enter an integer: 0
The value is zero

Lưu ý: Khi bạn dùng chuỗi các câu lệnh if như trên, các câu lệnh if sẽ có mối quan hệ với nhau như mình đã nói. Mối quan hệ đó như thế nào? Thứ nhất, đó là chỉ có một câu lệnh trong chuỗi đó được phép thực thi, còn lại sẽ bị bỏ qua hoặc không có câu lệnh nào được thực thi nếu các điều kiện đều fasle. Thứ hai, compiler sẽ xét các điều kiện từ trên xuống dưới. Nếu đến một điều kiện nào đó thoả mãn thì câu lệnh của điều kiện đó được thực thi và compiler sẽ không kiểm tra tiếp các điều kiện bên dưới nữa, nó sẽ thoát câu lệnh if. Ví dụ chương trình trên ta nhập 4 thì điều kiện đầu tiên x > 0 true nên nó thực thi câu lệnh tương ứng và bỏ qua hết các điều kiện dưới. Đây là lợi ích của chuỗi "if-else if-else" so với chuỗi "if-if", nếu ta dùng chuỗi "if-if" như chương trình đầu tiên của mục If-else thì chương trình sẽ mất thêm thời gian hơn so với "if-else" vì nó sẽ kiểm tra hết cả hai điều kiện cho dù điều kiện đầu tiên đúng. Lưu ý thứ ba, cách viết "if-else if-else" như trên chỉ dùng có trường hợp với mỗi điều kiện chỉ có duy nhất một câu lệnh cần thực thi, đối với trường hợp nhiều câu lệnh trong một điều kiện ta sẽ học ở bài sau.

Bài tập

Số nguyên tố (prime number) là những số chỉ có thể chia hết cho 1 và chính nó. Hãy viết một chương trình yêu cầu người dùng nhập một số nguyên chỉ có một chữ số (nhỏ hơn 10) có sử dụng hàm boolean và chuỗi câu lệnh if để chia trường hợp. Nếu người dùng nhập các số như 2, 3, 5, 7 thì in ra "The digit is prime", nếu không thì in ra "The digit is not prime".

#include <iostream>
 
bool isPrime(int x)
{
    if (x == 2)      // nếu người dùng nhập 2 thì số đó là số nguyên tố 
        return true;
    else if (x == 3) // nếu người dùng nhập 3 thì số đó là số nguyên tố 
        return true;
    else if (x == 5) // nếu người dùng nhập 5 thì số đó là số nguyên tố 
        return true;
    else if (x == 7) // nếu người dùng nhập 7 thì số đó là số nguyên tố 
        return true;
 
    return false;    // nếu không vào các trường hợp trên thì nó không là số nguyên tố
}
 
int main()
{
    std::cout << "Enter a single digit integer: ";
    int x {};
    std::cin >> x;
 
    if ( isPrime(x) )
        std::cout << "The digit is prime\n";
    else
        std::cout << "The digit is not prime\n";
 
    return 0;
}

Mình đoán là nhiều bạn sẽ viết thêm else return false; ở cuối chuỗi câu lệnh if. Các bạn làm vậy cũng đúng nhé, hai cách viết đều như nhau cả.
